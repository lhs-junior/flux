# Guide System User Guide

The Guide System helps you find exactly what you need to learn. Instead of searching through documentation, ask for what you want to learn and get guided tutorials, explanations, and resources.

## What is the Guide System?

The Guide System is your learning assistant. It helps you discover tutorials, find documentation, get step-by-step guidance, and learn how to use FLUX features effectively.

**Key benefits:**
- Find guides on any FLUX feature or concept
- Get step-by-step tutorials on how to do things
- Search with natural language (not keywords)
- Discover related guides and topics
- Learn at your own pace with structured guidance

## Main Operations

### Search Guides

Find guides on topics you want to learn:

```
You: "Show me guides on memory system"
FLUX: Returns all guides related to memory

Or: "How do I save and recall information?"
FLUX: Finds guides about memory operations

Or: "TDD workflow"
FLUX: Returns guides on test-driven development
```

### Get Tutorial

Start a step-by-step learning path:

```
You: "Give me a tutorial on using agents"

```

### Find Related Content

Discover connected topics:

```

FLUX: Recommends related guides (agents, planning, etc.)
```

### Get Quick Help

Find answers fast:

```
You: "How do I save information to memory?"

Or: "What's the difference between pending and in-progress tasks?"
```

## How to Use Guides Effectively

### Search for What You Need

**Natural language works:**
```
Good: "How do I organize information?"
Good: "Tell me about saving information"
Also: "Memory organization"

All find relevant guides
```

**Be specific if needed:**
```
Instead of: "Agents"
Try: "How do I delegate work to agents?"

Instead of: "Testing"
Try: "How do I use TDD to build features?"
```

### Follow Tutorials

**Tutorials are step-by-step:**

```
Each step includes:
1. What to do
2. Why you're doing it
3. What to expect
4. Tips and tricks

Follow at your own pace - no rush
```

**Try the examples:**

```
1. Read the explanation
2. Try the example yourself
3. See what happens
4. Move to next step

Hands-on learning sticks better
```

### Deep Dive into Guides

**Guides cover everything:**

```
Each feature guide includes:
- What it is and why use it
- Main operations (how to use)
- Use cases and examples
- Best practices
- Integration with other features
- Tips and tricks
- Troubleshooting
```

**Read what matters to you:**

```
Don't need to read everything

Just read:
- Overview (quick understanding)
- Use cases (see if it's relevant)
- Examples (see it in action)
- Skip the rest initially
```

### Learn as You Go

**You don't need to memorize everything:**

```
Better approach:
1. Learn the basics
2. Start using feature
3. When you get stuck, search guides
4. Learn what you need when you need it

This natural learning works well
```

## Troubleshooting

**Can't find what I'm looking for:**
- Try different search terms
- Break question into smaller parts
- Look at related guides for context
- Ask for "best practices" or "tutorials"

**Guide assumes I know something I don't:**
- Search for prerequisite topics
- Ask for explanation of specific concepts
- Look for beginner-level guides first

**Want to learn faster:**
- Follow structured tutorials (not just reading)
- Try examples while learning
- Take notes of important points
- Practice immediately after learning

**Don't remember something:**
- Search for it again (guides are always available)
- It's not about memorization
- Look it up when you need it
